# 投屏标题与观看历史设计

## 背景

投屏播放普通视频后，播放器 overlay 不应显示通用兜底文案“投屏视频”，而应显示投屏视频的真实标题。同时，投屏播放的视频应像应用内直接播放的视频一样，正常向 B 站上报播放历史，使其进入账号的最近观看记录。

## 目标

- 投屏普通视频时，播放器 overlay 显示视频标题，而不是“投屏视频”。
- 投屏普通视频时，继续走现有普通视频播放页的播放心跳/进度上报逻辑。
- 最近观看来源仍为 B 站历史接口，不新增本地投屏最近观看列表。

## 非目标

- 不为未登录或 B 站历史接口失败场景新增本地历史兜底。
- 不改变直播投屏标题逻辑，除非普通视频改动中需要保持同类兜底一致性。
- 不重构播放器、投屏订阅或历史页整体架构。

## 推荐方案

采用最小范围修复：

1. 投屏命令带有 `title` 时，普通视频播放对象直接使用该标题，播放器 overlay 显示该标题。
2. 投屏命令没有 `title` 时，避免把“投屏视频”作为最终用户可见标题；即使投屏命令已经带有 `cid`，也需要新增标题解析路径，通过视频详情查询补齐真实标题。
3. 投屏普通视频仍进入 `PlayerPage`，复用普通视频播放的心跳/进度上报路径，让 B 站服务端历史记录自然更新。

## 数据流

```text
投屏服务收到 play 命令
  -> App castSubscribe 回调构造 playerVideo
  -> PlayerPage 加载视频与详情
  -> overlay 显示 playerVideo.title 或详情标题
  -> 播放过程中执行现有 B 站心跳/进度上报
  -> B 站历史接口返回该视频
  -> HistoryPage 展示最近观看
```

## 标题显示规则

普通视频投屏标题优先级：

1. `command.title`：投屏端已提供标题时直接使用。
2. `getVideoInfo` 返回的标题：投屏端未提供标题时，播放页加载详情后补齐。
3. 无标题状态：如果投屏端未提供标题且详情接口失败，overlay 标题行应为空或隐藏，不显示“投屏视频”这类误导性通用标题。

当前代码中，`PlayerPage` 只在缺少 `cid` 时调用 `getVideoInfo`。实现时不能依赖这条现有路径覆盖所有投屏场景，因为常见投屏命令会同时带有 `cid` 但不一定带有 `title`。实现需要显式处理 `fromCast && !title` 的情况，例如：

- 对投屏来源的视频始终请求一次详情用于补齐标题，即使已经有 `cid`；或
- 增加独立标题解析逻辑，仅在 `fromCast && !title` 时请求详情并更新 overlay 标题。

## 最近观看规则

- 不新增本地存储项。
- 不修改 `HistoryPage` 的数据来源；它继续调用 B 站历史接口。
- 投屏普通视频必须复用普通视频播放器中的历史/心跳上报逻辑。
- 验收以 B 站账号历史记录是否出现该视频为准。

本地播放进度 `resume_progress` 与最近观看是两个不同概念。投屏视频可以继续参与现有播放进度恢复逻辑，但本次不新增“本地最近观看”列表，也不把 `resume_progress` 当作最近观看数据源。

## 已知限制

- 当前投屏普通视频对象的 `owner.name` 可能为空，因此 overlay 副标题可能不显示 UP 主信息。本次目标只要求标题正确，不要求补齐作者信息。
- 未登录账号无法写入 B 站历史记录；这是现有 B 站账号历史机制限制。

## 验收标准

- 投屏命令包含 `title: "示例视频标题"` 时，播放器 overlay 显示“示例视频标题”，不显示“投屏视频”。
- 投屏命令不包含 `title`、但包含 `cid`，且视频详情接口返回标题时，overlay 最终显示详情标题。
- 投屏命令不包含 `title`，且视频详情接口失败时，overlay 不显示“投屏视频”。
- 投屏播放普通视频一段时间后，B 站历史记录能看到该视频。
- 应用“最近观看”页刷新后，能通过 B 站历史接口展示该投屏播放记录。

## 测试建议

- 单元/渲染测试：模拟投屏 `play` 命令带标题，断言播放器标题为命令标题。
- 单元/渲染测试：模拟投屏 `play` 命令不带标题但带 `cid`，视频详情返回标题，断言 overlay 最终更新为详情标题。
- 单元/渲染测试：模拟投屏 `play` 命令不带标题且详情接口失败，断言 overlay 不显示“投屏视频”。
- 回归测试：确认投屏普通视频仍触发普通视频播放页已有的心跳/进度上报调用；自动化测试可优先验证心跳调用，手动测试再验证服务端历史最终出现。
- 手动验证：在已登录账号上投屏播放视频，等待心跳上报后刷新最近观看页。

## 风险与处理

- 如果投屏端不传标题且视频详情接口失败，overlay 可能无标题。该行为优于展示错误的通用标题。
- 如果 B 站服务端历史写入有延迟，最近观看页可能需要刷新或等待一段时间。
- 如果用户未登录，B 站历史不会更新；这是现有账号历史机制限制，不在本次范围内解决。
